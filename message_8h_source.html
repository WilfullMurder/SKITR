16 * The 'params' field is optional and can be an array (for positional parameters) or an object (for named parameters).
17 * For simplicity, we only support positional parameters as a vector of Value.
18 * The 'id' field is optional. If it is present, the message is a request and a response with the same 'id' must be sent back.
19 * If it is absent, the message is a notification and no response should be sent.
20 */

23 std::vector<Value> params; ///< Positional parameters. Can be empty if method takes no parameters.
24 std::optional<Value> id; ///< present => response expected; absent => notification
25
26 bool isNotification() const noexcept { return !id.has_value(); }

39 /**
40 * Checks if the response is valid according to JSON-RPC 2.0 rules: exactly one of 'result' or 'error' must be present, and 'id' must be present.
41 * @return true if the response is valid, false otherwise
42 * @note This does not check if 'id' matches any request, as that is the responsibility of the caller.
